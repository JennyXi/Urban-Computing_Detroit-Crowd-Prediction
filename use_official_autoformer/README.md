## use_official_autoformer

This folder runs the **official** [thuml/Autoformer](https://github.com/thuml/Autoformer.git) training code on the **2025 Detroit crowd datasets** generated by this repo.

### What it uses (your existing outputs)

- **Weekly (2025, 52 weeks)**: `../data/autoformer_weekly_preprocessed.csv`
- **Hourly (2025, 8736 hours)**: `../data/autoformer_hourly_preprocessed.csv`

Both files already follow the `Dataset_Custom` convention used by Autoformer-style repos:
- first column: `date`
- last column: `OT` (target)
- middle columns: numeric covariates/channels

### One-time environment check

These scripts assume you can run:

```powershell
.\.venv\Scripts\python.exe -c "import torch, sklearn, pandas"
```

If missing packages, install them into your existing `.venv`:

```powershell
.\.venv\Scripts\pip.exe install -r .\use_official_autoformer\third_party\Autoformer\requirements.txt
```

### Train (Weekly 2025)

```powershell
powershell -ExecutionPolicy Bypass -File .\use_official_autoformer\runs\train_weekly_2025.ps1
```

### Train (Hourly 2025)

```powershell
powershell -ExecutionPolicy Bypass -File .\use_official_autoformer\runs\train_hourly_2025.ps1
```

### Export predictions (CSV with dates)

After training, export the **test-set** predictions into a CSV aligned with `date`.

Weekly (defaults match your current weekly config `12 -> 4`):

```powershell
.\.venv\Scripts\python.exe .\use_official_autoformer\export_predictions.py --setting latest --setting-prefix detroit_2025_weekly_
```

Outputs:
- `use_official_autoformer/outputs/<setting>_pred.csv`

CSV columns:
- `sample_id`: test window index
- `window_start`: the first date of the input window
- `horizon`: 1..pred_len
- `date`: the predicted week start date for that horizon
- `y_true`, `y_pred`: target (`OT`) on the original scale

To export rolling forecasts across the **whole year** (not only the test split), use:

```powershell
.\.venv\Scripts\python.exe .\use_official_autoformer\export_predictions.py --scope all --setting latest --setting-prefix detroit_2025_weekly_
```

Note: with `seq_len=12`, the earliest predictable week is the 13th week (you need 12 weeks of history).

If you want forecasts covering **Janâ€“Dec 2025**, you must ensure the input CSV includes **earlier history** (e.g. 2024).
One option is to train on the longer weekly file `../data/autoformer_weekly.csv` (multi-year).

The exporter also writes an Excel-friendly aggregated file:
- `<setting>_<scope>_pred_by_date.csv` (one row per `date`, with `y_pred_mean` / `y_pred_last`)

### Notes / gotchas

- **No data copy**: we pass `--root_path` pointing at your repo `data/` directory directly.
- **CPU training**: your `.venv` currently has `torch ... +cpu`; training will be slower but should run.
- **Autoformer split**: `Dataset_Custom` uses **70/10/20** time-ordered split internally.
- **Dimensions**: scripts auto-detect `enc_in/dec_in/c_out` from the CSV column count.
